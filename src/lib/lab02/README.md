<h1>collection.py<h1>

> В collection.py представлена реализация коллекции House и ее методов

## Импорты:

### Из Лаб. №1:

* `Apartment` - Импорт структуры Apartment

### Из validation.py:

* `validate_class_apartment` - Ф-ия валидации принадлежности объекта классу Apartment.

* `validate_apartment_address` - Ф-ия валидации атрибута `address` класса apartment.

* `validate_unique_id` - Ф-ия валидации уникального идентификатора (ID).

## Коллекция House:

### Атрибуты:

* `apartments` - Контейнер объектов коллекции House (type: list).

### Dunder-методы:

* `__len__` - Красивый вывод списка коллекции House.

* `__iter__` - Преобразование обычного списка коллекции House в итератор.

* `__getitem__` - Обращение к коллекции как к списку объектов и реализация доступа к ним по индексу.

### Инкапсуляция:

Геттер `@property` для атрибута `apartments`, возвращающий этот же атрибут.

### Методы:

* `remove` - Удаления объекта из коллекции House (Принимает на вход объект класса Apartment).

* `add` - Добавление объекта в коллекцию House (Принимает на вход объект класса Apartment).

* `get_all` - Красивый вывод всех объектов списка коллекции House.

* `find_by_adress` - Поиск объекта коллекции House по адресу (Принимает на вход сам адрес (type: str)).

* `remove_by_index` - Удаления объекта по индексу списка коллекции House (Принимает на вход index (type: int)).

> Все методы `sort_*****` принимают на вход значение, определяющее порядок сортировки (type: bool).

* `sort_by_name` - Сортировка объектов коллекции House по имени.

* `sort_by_price` - Сортировка объектов коллекции House по цене.

* `sort_by_squares` - Сортировка объектов коллекции House по площади.

* `get_expensive` - Вывод самых дорогих квартир коллекции House (Если цена квартиры >= сред. цене по коллекции - Квартира считается дорогой).

* `get_available` - Вывод доступных для покупки квартир (Если атрибут `is_available` класса Apartment - True).
